# سجل مشكلات AutoHR

آخر تحديث: 2026-06-06

هذا الملف هو السجل المحلي المعتمد للمشكلات والعيوب الفنية في المشروع. عند اكتشاف
مشكلة جديدة، أضفها إلى الجدول ثم أنشئ لها قسمًا تفصيليًا باستخدام القالب الموجود
في نهاية الملف.

## الحالات والأولويات

| القيمة | المعنى |
|---|---|
| مفتوحة | المشكلة مؤكدة ولم تعالج بعد |
| قيد المعالجة | بدأ العمل على المشكلة |
| معالجة | نُفذ الحل وتم التحقق منه |
| مؤجلة | المشكلة معروفة لكن لن تعالج حاليًا |
| حرجة | خطر أمني أو فقدان بيانات أو توقف كامل |
| عالية | تعطل وظيفة أساسية |
| متوسطة | خلل مؤثر مع وجود حل بديل |
| منخفضة | تحسين أو تحذير لا يعطل الاستخدام |

| AHR-002 | وظائف تصدير PDF وExcel وطباعة بطاقة الموظف غير منفذة | عالية | مفتوحة | 2026-06-06 |

## تفاصيل المشكلات

### AHR-001: حماية الجلسة ومسارات المرفقات

- **الحالة:** معالجة
- **الأولوية:** حرجة
- **الوصف:** كانت حدود الثقة بين الواجهة والخلفية بحاجة إلى حماية؛ يجب ألا تعتمد
  الأوامر الحساسة على اسم مستخدم ترسله الواجهة، ويجب منع المسارات المطلقة أو التي
  تحتوي على `..` عند حذف المرفقات.
- **المعالجة الحالية:** تستخدم أوامر Tauri الحساسة `SessionState` لتحديد المستخدم
  المسجل، وتتحقق الدالة `safe_relative_path` من مسارات المرفقات.
- **الدليل:** `src-tauri/src/lib.rs` و`src-tauri/src/files.rs`.
- **التحقق بتاريخ 2026-06-06:** نجح الأمر `cargo test --workspace`.
- **ملاحظة:** لا يوجد سجل Git يثبت تاريخ ظهور المشكلة أو التغيير الذي عالجها؛
  تسجيلها كمعالجة مبني على مراجعة الحالة الحالية للكود.

### AHR-002: وظائف التصدير والطباعة غير منفذة

- **الحالة:** مفتوحة
- **الأولوية:** عالية
- **الوصف:** أوامر تصدير الموظفين إلى PDF وExcel وطباعة بطاقة الموظف تعيد نصًا
  فارغًا دون تنفيذ الوظيفة المطلوبة.
- **الدليل:** الدوال `cmd_export_employees_pdf` و`cmd_export_employees_excel`
  و`cmd_print_employee_card` في `src-tauri/src/lib.rs`.
- **الأثر:** تظهر الوظائف في جسر الواجهة، لكنها لا تنتج ملفًا أو نتيجة مفيدة.
- **معيار الإغلاق:** تنفيذ الوظائف، التحقق من الملفات الناتجة، وإضافة اختبارات
  تغطي حالات النجاح والفشل.

### AHR-003: عدم وجود اختبارات آلية فعلية

- **الحالة:** مفتوحة
- **الأولوية:** متوسطة
- **الوصف:** ينجح `cargo test --workspace` لكنه يشغّل صفر اختبارات، ولا توجد
  أوامر اختبار للواجهة في `package.json`.
- **الأثر:** قد تمر الانحدارات الوظيفية أو الأمنية دون اكتشاف.
- **معيار الإغلاق:** إضافة اختبارات مركزة للجلسة والصلاحيات ومسارات الملفات
  وعمليات الموظفين، وإضافة أمر اختبار للواجهة عند الحاجة.

### AHR-004: تحذيرات تقسيم الحزم في Vite

- **الحالة:** مفتوحة
- **الأولوية:** منخفضة
- **الوصف:** يبني Vite المشروع بنجاح، لكنه يحذر من استيراد وحدات Tauri نفسها
  بشكل ثابت وديناميكي، لذلك لن تنقل الاستيرادات الديناميكية هذه الوحدات إلى حزم
  منفصلة.
- **الأثر:** لا يعطل البناء، لكنه يجعل تقسيم الحزم أقل فاعلية ويزيد الغموض في
  استراتيجية التحميل.
- **التحقق بتاريخ 2026-06-06:** نجح `npm run build` مع ظهور التحذيرات.
- **معيار الإغلاق:** توحيد نمط استيراد وحدات Tauri أو توثيق قبول التحذير.

### AHR-005: غياب سجل Git

- **الحالة:** مفتوحة
- **الأولوية:** متوسطة
- **الوصف:** المجلد الحالي ليس مستودع Git، لذلك لا يمكن معرفة تاريخ التغييرات أو
  ربط المشكلات بالإصلاحات ومراجعتها.
- **الأثر:** يصعب تتبع الانحدارات، مقارنة التعديلات، والتراجع عن تغيير خاطئ.
- **معيار الإغلاق:** تهيئة مستودع Git، إنشاء أول commit موثق، واعتماد رسائل commit
  تشير إلى معرّفات المشكلات مثل `AHR-002`.

```markdown
- **الأولوية:** منخفضة / متوسطة / عالية / حرجة

- **الوصف:** ما السلوك الخاطئ؟

- **معيار الإغلاق:** شروط واضحة لاعتبار المشكلة معالجة.
```
